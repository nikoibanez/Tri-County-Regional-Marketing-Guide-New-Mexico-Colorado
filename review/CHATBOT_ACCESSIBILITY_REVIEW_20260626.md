# Directory Assistant Accessibility Review

Date: 2026-06-26

Scope: the global "Ask directory" assistant generated by `tools/build_netlify_deep_guide.py` and included on every public page in `dist/tri-county-netlify-guide-deep`.

## Changes Implemented

- Replaced the custom ARIA dialog/focus-trap pattern with a native `<dialog>` and `showModal()` flow, following the Ponytail/native-platform approach.
- Converted the assistant panel to a native `<dialog>` with `aria-labelledby` and `aria-describedby`.
- Added `aria-haspopup="dialog"` to the opener and preserved `aria-expanded` state.
- Added an assistive-technology hint explaining that results update after typing pauses and that Escape closes the panel.
- Linked the search input to the results region with `aria-controls`.
- Added a polite, atomic status region for result-count announcements.
- Changed results into a semantic `role="list"` with each result as `role="listitem"`.
- Added accessible labels to repeated "Open source" and "Search full directory" result links so links-list navigation is more meaningful.
- Used native dialog focus containment and Escape handling, with a small close/focus-return wrapper.
- Debounced type-ahead results to reduce excessive live-region chatter.
- Increased close and suggested-search button touch targets.
- Added `scripts/audit_ui_accessibility.py`, a no-dependency static audit for generated accessibility regressions.

## Current Accessibility Posture

The assistant is now substantially stronger for:

- Keyboard-only navigation.
- Screen-reader naming and context.
- Result announcements.
- Mobile/touch target usability.
- Predictable close behavior.
- Reduced cognitive load from less frequent live-region updates.

## Remaining Manual QA

These need a real browser and assistive technology pass before a public accessibility claim:

- Screen-reader behavior in NVDA, JAWS, VoiceOver, or Narrator.
- Keyboard tab order in Chrome, Edge, Firefox, and Safari.
- Mobile screen-reader behavior on iOS and Android.
- Color contrast against the live animated/transparent background in final deployed context.
- Whether the assistant and music controls feel crowded at narrow viewport heights.

## Notes

The assistant is a deterministic client-side directory search over the guide data. It does not call an AI API or generate new facts. It should continue to present source links and update reminders rather than answering as an authority.

Ponytail note: the best improvement was deletion, not expansion. Native `<dialog>` removed the need for custom Tab trapping while improving browser semantics.
